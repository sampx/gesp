<task type="auto">
  <name>Fix Tailwind v4 CSS import in globals.css</name>
  <files>apps/web/src/app/globals.css</files>
  <action>
The root cause of Gap 1 (login page "几乎无样式"): Tailwind v4 utility classes are not generated because `globals.css` is missing the core `@import "tailwindcss"` directive.

The file currently starts with `@import "tw-animate-css"` but has no `@import "tailwindcss"`. In Tailwind v4, the `@tailwindcss/postcss` plugin processes `@import "tailwindcss"` to inject all utility classes, base styles, and the theme system. Without it, only raw CSS custom properties are defined but no utility classes exist — so `className="min-h-screen flex items-center..."` produces no styles.

Fix: Add `@import "tailwindcss"` as the FIRST line of `apps/web/src/app/globals.css`, before the existing `@import "tw-animate-css"`:

```css
@import "tailwindcss";
@import "tw-animate-css";
```

This is the standard Tailwind v4 setup — `@import "tailwindcss"` loads the framework, and `@import "tw-animate-css"` adds animation utilities. The rest of the file (`@custom-variant`, `@theme inline`, `@layer base`) remains unchanged.

Do NOT change anything else in the file.
  </action>

    <automated>head -2 apps/web/src/app/globals.css | grep -q "tailwindcss" && echo "PASS: @import tailwindcss found"</automated>
  </verify>
