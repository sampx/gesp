# Phase 2 — UI Design Contract

> Visual and interaction contract for GESP 知识库 + 双端骨架 phase. Generated by wsf-ui-researcher, verified by wsf-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | default (init during Phase 2 execution) |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | Lucide React (bundled with shadcn/ui) |
| Font (Student) | Inter — Google Fonts, weights 400/600 |
| Font (Admin) | System font stack — Apple System, Segoe UI, Roboto, sans-serif, weights 400/600 |

> **Dual-style strategy:** Single shadcn/ui component library, two CSS variable themes. Student theme applies warm/playful variables; Admin theme applies cool/professional variables. Theme applied via `<html data-theme="student|admin">` at route group layout level.

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing, badge padding |

| xl | 32px | Layout gaps, sidebar width padding |

Exceptions: None — all spacing uses 8-point grid (multiples of 4px).

## Layout Specifications

### Login Page (`/login`)
- Centered card layout, max-width 400px
- Three role cards horizontal (desktop) / vertical stack (mobile `<640px`)
- Selected card: accent border + slight scale(1.02)
- Below cards: username/password form with `<Button>` submit
- Footer text: "选择角色后输入账号密码登录"
- Background: subtle gradient or illustration area (defer illustration to Phase 6)

### Student Layout (`(student)/layout.tsx`)
- **Mobile-first:** Bottom tab bar with 4 items (首页/测评/学习/练习) + lucide icons
- **Header:** Minimal — avatar + username left, notification icon right
- **Content:** `<main>` with padding-md (16px), max-width 768px centered
- **Placeholder pages:** Card with emoji icon + title + "Phase X 见！" description

### Admin Layout (`(admin)/layout.tsx`)
- **Desktop-first:** Left sidebar (width 240px, collapsible to 64px icon-only)
- **Sidebar items:** Dashboard, 知识库 (expandable: 知识点/教案/练习题/真题), 学员管理, 数据分析, 设置
- **Header:** Breadcrumb left, user avatar + dropdown right
- **Content:** `<main>` with padding-lg (24px), full-width within sidebar
- **Knowledge base pages:** DataTable primary, Sheet for detail/edit

## Interaction Patterns

### Role Selection (Login)
1. User sees 3 cards: 学员🎮 / 教员📚 / 管理员⚙️
2. Click/tap card → card highlights with accent border + slight elevation
3. Form appears below → fill username + password → click 登录
4. POST `/api/auth/login` → on success: redirect by role
5. On failure: Toast error "用户名或密码错误" (no user enumeration leak)

### Knowledge Base CRUD (Admin)
1. List view: DataTable with columns (名称, 级别, 主题, 更新时间, 操作)
2. Filter bar: Select dropdowns for level (1-8), block (选择/循环/数组...), tag
3. Search: Command palette (Cmd+K) or search input in header
4. Create: Button "添加知识点" → Dialog/Sheet with react-hook-form + Zod
5. Edit: Row action → Sheet slide-in with pre-populated form
6. Delete: Row action → Dialog "确认删除知识点？此操作不可撤销" → destructive red button
7. Markdown content: Textarea with live preview (react-markdown rendering)

### Navigation
- **Student:** Bottom tab bar selected state uses accent color + filled icon variant
- **Admin:** Sidebar selected state uses accent left-border + accent background tint
- **Auth redirect:** Middleware checks session cookie → valid: proceed; invalid: redirect `/login`
