<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → assessment history APIs | Student-facing UI consumes ownership-scoped session metadata |
| Delete action → backend | UI can remove persisted assessment records |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09-01 | Information Disclosure | session history list | mitigate | Render only server-returned current-user sessions; do not cache/share across users. |
| T-03-09-02 | Tampering | delete action | mitigate | Use backend ownership-checked delete endpoint; on client, remove rows only after successful response or restore on failure. |
| T-03-09-03 | Repudiation | resume flow | mitigate | Call backend `resumeAssessment(token)` before navigation so ellamaka session restoration is explicit and logged server-side. |
</threat_model>
