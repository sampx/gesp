<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student session cookie → /sessions endpoints | Only the authenticated student should see and delete their own history |
| backend readiness state → frontend SSE consumer | System events must reflect real lock/completion state, not inferred guesses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08-01 | Information Disclosure | GET /sessions | mitigate | Scope all session history queries by authenticated `student_id` and return only current user's sessions. |
| T-03-08-02 | Tampering | DELETE /sessions/:sessionId | mitigate | Require ownership check before deleting session rows, answer rows, timers, and projector state. |
| T-03-08-03 | Repudiation | question_ready / assessment_done events | mitigate | Emit system events only from definitive backend state transitions (`lockQuestion`, completion), not from speculative frontend timing. |
| T-03-08-04 | Denial of Service | auto-select timer | mitigate | Restore 10s fallback and shared readiness path so sessions cannot stall for 30s due to timer drift. |
</threat_model>
