**1. JWT Token utilities:**
```typescript
import { sign, verify } from "hono/jwt";

const JWT_SECRET = process.env.JWT_SECRET || "gesp-assessment-secret-dev";
const TOKEN_EXPIRY_SEC = 2 * 60 * 60; // 2 hours

interface TokenPayload {
  assessment_session_id: string;
  student_id: string;
  course_id: string;
  exp?: number;
}

export async function generateToken(payload: Omit<TokenPayload, "exp">): Promise<string> {
  return await sign({ ...payload, exp: Math.floor(Date.now() / 1000) + TOKEN_EXPIRY_SEC }, JWT_SECRET);
}

export async function verifyToken(token: string): Promise<TokenPayload> {
  const decoded = await verify(token, JWT_SECRET);
  return decoded as unknown as TokenPayload;
}
```

Implementation: Count correct answers. If `correct >= thresholdUp` → advance (level+1, bounded at 8). If `correct <= thresholdDown` → retreat (level-1, bounded at 1). ## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Assessment token → Service | JWT token crosses trust boundary from client to backend; must be verified before use |
| Service → SQLite | Service writes user-supplied answers to DB; SQL injection risk |
| Environment → Service | JWT_SECRET read from environment; exposure risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | generateToken / JWT_SECRET | mitigate | Read JWT_SECRET from `process.env.JWT_SECRET`, never hardcode in source. Dev fallback value only for local dev — document that production MUST set env var. |
| T-03-02 | Tampering | verifyToken | mitigate | Verify JWT signature using hono/jwt `verify()`. Reject tokens with expired `exp` claim. Token payload includes student_id — validate that requesting student matches token's student_id in route layer (Plan 03). |
| T-03-03 | Elevation of Privilege | createAssessmentSession | mitigate | Session creation requires valid session auth (StudentAuth middleware in route layer). student_id comes from authenticated session context, not user input. |
| T-03-04 | Injection | getCandidates / SQL queries | mitigate | All queries use Drizzle ORM parameterized queries — no raw SQL string concatenation. `inArray()`, `eq()`, `notInArray()` are all parameterized. |
| T-03-05 | Information Disclosure | getCandidates / short_summary | mitigate | Return only `content.slice(0, 100) + "..."` — never full question content in candidate list. Full content only returned when question is locked via `lockQuestion`. |
</threat_model>
