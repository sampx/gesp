<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted session row → runtime lock | 服务器重启后，DB 持久态要恢复运行态激活题目 |
| historical token → resume/progress endpoints | 老 token 进入继续测评路径，需要区分合法恢复与非法越权 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11-01 | Tampering | `current_question_id` session field | mitigate | `/submit` 与 `/next-question` 通过 token→session→current_question_id 校验题目归属，不接受任意 question_id。 |
| T-03-11-02 | Denial of Service | memory-lock loss after restart | mitigate | 将 active question 写入 `assessment_sessions.current_question_id`，并在运行态缺失时回退 DB。 |
| T-03-11-03 | Elevation of Privilege | resume with old token | mitigate | age check removal同时保留 token existence + session ownership + `StudentAuth()` resume gate，completed 会话仍禁止继续答题。 |
| T-03-11-04 | Information Disclosure | report progress payload | accept | `knowledge_stats` 仅返回当前 token 对应 session 的聚合统计，不暴露他人数据。 |
</threat_model>
