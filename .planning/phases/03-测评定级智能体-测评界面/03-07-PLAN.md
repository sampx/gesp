<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| assessor tool call → backend progress API | Agent relies on token-scoped progress data to make termination decisions |
| coding score tool → assessment_answers | AI-generated score crosses into persisted grading data |
| evaluation endpoint → session status | Final agent action can end the assessment lifecycle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07-01 | Tampering | update_answer_score tool | mitigate | Verify token/session/question ownership before updating a coding-answer row; only update an existing row in the same session. |
| T-03-07-02 | Information Disclosure | query_progress tool | mitigate | Use token-scoped progress only; return aggregate counts/timing, never full question content or answers. |
| T-03-07-03 | Denial of Service | SSE logging | mitigate | Lower high-frequency event logs to trace/suppress to prevent log volume explosion during normal assessments. |
| T-03-07-04 | Repudiation | completed session transition | mitigate | Persist `completed_at` and final status at `/evaluate`, so termination is auditable and subsequent APIs cannot pretend the session is still active. |
</threat_model>
