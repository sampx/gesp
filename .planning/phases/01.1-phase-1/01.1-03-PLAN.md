**Problem:** Test 3 (lines 57-65) tests password validation with "short" password. CR-01 fix moved password check BEFORE DB query (auth.service.ts line 13-14), so `findFirst` is never called. The mock `mockResolvedValueOnce(undefined)` at line 59 remains queued → pollutes Test 4's mock queue.

**Fix:** Delete the unnecessary `findFirst` mock from Test 3 since that test never reaches the DB query.

Change lines 57-65 from:
```typescript
it("should preserve password validation error message (not security-sensitive)", async () => {
  // Mock: no existing user
  (db.query.users.findFirst as any).mockResolvedValueOnce(undefined);

  const result = await registerUser("newuser", "short", "Test");

  expect(result.success).toBe(false);
  expect(result.error).toContain("Password must be at least 6 characters");
});
```

To:
```typescript
it("should preserve password validation error message (not security-sensitive)", async () => {
  // No mock needed: password check happens BEFORE DB query (CR-01 fix)
  // registerUser returns early without calling findFirst

  const result = await registerUser("newuser", "short", "Test");

  expect(result.success).toBe(false);
  expect(result.error).toContain("Password must be at least 6 characters");
});
```

**Rationale:** Test 3 validates password length. Password "short" (5 chars) triggers early return at auth.service.ts:14. No DB call, so no mock needed. Removing the mock prevents queue pollution.

<task type="auto">
  <name>Task 2: Fix JavaScript newline escape in debug.ts</name>
  <files>packages/backend/src/routes/debug.ts</files>
  <action>
**Problem:** Line 97 in the template literal has `document.createTextNode('\n')`. TypeScript template literal preserves the escape sequence as a REAL newline character when rendering to HTML. Browser receives broken JavaScript:
```javascript
logEl.appendChild(document.createTextNode('
')); // SyntaxError: unterminated string
```
This breaks all onclick handlers → buttons unresponsive.

**Fix:** Change `'\n'` to `'\\n'` (double-escaped) so the browser receives the escape sequence for newline, not a literal newline.

Change line 97 from:
```typescript
logEl.appendChild(document.createTextNode('\n'));
```
To:
```typescript
logEl.appendChild(document.createTextNode('\\n'));
```

**Alternative:** Could also use `'\u000a'` (Unicode escape), but `'\\n'` is more readable and idiomatic.

**Verification:** After fix, browser DevTools console should show no SyntaxError, and clicking Register/Login buttons should trigger fetch calls visible in Network tab.
