### 2. Session TTL 双重验证逻辑（D-R4）

**现状分析：**

当前 `validateSession` 实现（session.ts:44-72）：

```typescript
export async function validateSession(c: Context, next: Next): Promise<Response | void> {
  const sessionId = getCookie(c, "session_id");           // 1. 解析 cookie
  if (!sessionId) return unauthorized(c, "No session cookie");

  const session = await db.query.sessions.findFirst({    // 2. 查 DB
    where: eq(sessions.id, sessionId),
    with: { user: true },
  });

  if (!session) {                                         // 3. Session 不存在
    deleteCookie(c, "session_id");
    return unauthorized(c, "Session not found");
  }

  if (session.expires_at < new Date()) {                  // 4. TTL 过期校验
    await db.delete(sessions).where(eq(sessions.id, sessionId));
    deleteCookie(c, "session_id");
    return unauthorized(c, "Session expired");
  }

  c.set("user", session.user);                            // 5. 注入 context
  c.set("session", session);
  return next();
}
```

**已实现的验证流程：**
1. Cookie 解析 → 2. DB 查询验证 → 3. 过期时间比较 → 4. 注入 context

**缺失项：**
- ❌ Session ID 使用 `crypto.randomUUID()`（Line 21），不符合 D-R4 的 `crypto.getRandomValues()` 要求
- ❌ 无滑动过期机制（可选，REVIEWS.md 建议）
- ❌ 无 Session 清理定时任务（REVIEWS.md [MEDIUM]，已 Deferred）

**修复方案：**

```typescript
// 改用 crypto.getRandomValues 生成 32 字节随机值
function generateSessionId(): string {
  const bytes = new Uint8Array(32);
  crypto.getRandomValues(bytes);
  return Buffer.from(bytes).toString('base64url');
}

// 替换 createSession 中的 crypto.randomUUID()
const sessionId = generateSessionId();
```

### 3. 认证接口安全策略（D-R5, D-R6）

**现状分析：**

**注册接口（auth.service.ts:7-37）：**
```typescript
// 问题：错误消息泄露用户存在性
if (existing) {
  return { success: false, error: "Username already exists" };  // ❌ 泄露
}
```

**登录接口（auth.service.ts:39-61）：**
```typescript
// 已实现模糊化
if (!user) {
  return { success: false, error: "Invalid username or password" };  // ✅ 正确
}
if (!validPassword) {
  return { success: false, error: "Invalid username or password" };  // ✅ 正确
}
```

**修复方案：**

注册接口错误消息统一模糊化：
```typescript
if (existing) {
  return { success: false, error: "Registration failed. Please try different credentials." };
}
```

**注意：** 当前实现已用 `findFirst` 查询检查 uniqueness，这是正确的。问题仅在于错误消息泄露。

### 4. 默认密码安全策略（D-R7）

**现状分析：**

当前 `admin.seed.ts` 实现：
```typescript
const password = process.env.ADMIN_PASSWORD || "admin123";  // ❌ 无生产环境保护
console.log("WARNING: Change default password after first login in production!");
```

**问题：**
- 生产环境允许使用默认密码
- 仅输出警告，不阻止启动

**修复方案：**

```typescript
export async function seedAdmin(): Promise<void> {
  const existingRoot = await db.query.users.findFirst({
    where: eq(users.role, ROLE.ROOT),
  });
  if (existingRoot) {
    console.log("Admin already exists, skipping seed");
    return;
  }

  const password = process.env.ADMIN_PASSWORD;
  
  // 生产环境强制要求设置密码
  if (process.env.NODE_ENV === 'production' && !password) {
    throw new Error("ADMIN_PASSWORD must be set in production environment");
  }

  // 开发环境允许默认值，但输出强烈警告
  const finalPassword = password || "admin123";
  if (!password) {
    console.error("⚠️ WARNING: Using default password 'admin123'. CHANGE IMMEDIATELY!");
    console.error("⚠️ Set ADMIN_PASSWORD environment variable for security.");
  }

  // ... 继续创建 admin
}
```

**验证项设计：**

| Phase | 验证项 | 状态 |

## Validation Architecture (Nyquist)

### What to Validate

1. **Schema 约束验证**
   - expires_at 索引存在
   - username UNIQUE 约束存在
   - user_id ON DELETE CASCADE 存在

2. **Session 安全验证**
   - Session ID 为 32 字节 base64url 编码（非 UUID）
   - TTL 双重验证：cookie + DB expires_at

3. **认证接口安全验证**
   - 注册失败：错误消息不泄露用户存在性
   - 登录失败：统一模糊化消息

4. **默认密码安全验证**
   - 生产环境无 ADMIN_PASSWORD：拒绝启动
   - 开发环境默认密码：输出警告

### How to Validate

**自动化测试：**
```typescript
// Session ID 生成测试
it("should generate 32-byte base64url session ID", () => {
  const id = generateSessionId();
  expect(id.length).toBe(43); // 32 bytes -> 43 base64url chars
  expect(id).toMatch(/^[A-Za-z0-9_-]+$/);
});

// 注册错误消息测试
it("should not leak username existence", async () => {
  const res = await registerUser("existing", "password", "name");
  expect(res.error).not.toContain("already exists");
  expect(res.error).toContain("failed");
});

// 生产环境密码测试
it("should reject startup without ADMIN_PASSWORD in production", async () => {
  process.env.NODE_ENV = "production";
  delete process.env.ADMIN_PASSWORD;
  await expect(seedAdmin()).rejects.toThrow("ADMIN_PASSWORD must be set");
});
```

**手动验证：**
- 启动 backend，访问 `/debug`
- 依次执行：注册 → 登录 → 查看用户 → 登出
- 检查每个步骤的响应
