## Interaction Patterns

### Registration Flow

1. User navigates to `/register` (from login page link or direct URL)
2. Two role cards displayed: 学员 🎮 / 教员 📚 (管理员不可自助注册)
3. Click/tap card → card highlights with accent border
4. Fill username (3-20 chars) + password (6+ chars) + confirm password
5. Client-side validation in real-time (Zod schema)
6. Submit → Server Action calls `POST /api/auth/register`
7. **Success:** Auto-login + redirect to `/student/dashboard` (学员) or `/admin/dashboard` (教员)
8. **Failure:** Toast error (模糊化消息防止枚举)

### Login → Register Navigation

1. Login page bottom: "没有账号？去注册" link → navigates to `/register`
2. Register page bottom: "已有账号？去登录" link → navigates to `/login`

### Logout Flow

1. Student: Click avatar dropdown → "登出" | Admin: Click sidebar "登出" button
2. Server Action calls `POST /api/auth/logout`
3. Backend clears session cookie
4. Redirect to `/login`
5. **No confirmation dialog** (per D-07)

### Password Change Flow

1. User navigates to `/student/settings` or `/admin/settings`
2. Fill 旧密码 + 新密码 + 确认新密码
3. Client-side Zod validation: 新密码 ≥ 6 chars, 确认密码 match
4. Submit → Server Action calls `POST /api/auth/change-password`
5. **Success:** Toast "密码修改成功", keep current session (per D-10)
6. **Failure (wrong old password):** Inline error "旧密码不正确"

### Admin Add User Flow

1. Admin clicks "添加用户" button on `/admin/students`
2. `<AddUserDialog>` opens with role Select + username + password + confirm password
3. Role options: 学员 / 教员
4. Submit → `POST /api/admin/users`
5. **Success:** Toast "用户 {username} 创建成功", dialog closes, table refreshes
6. **Failure:** Toast error

### Admin Toggle User Status Flow

1. Admin clicks "禁用" or "启用" button in table row
2. `PUT /api/admin/users/:id/status` called
3. **Success:** Toast "已禁用/启用用户 {username}", table refreshes with updated badge
4. Button toggles: 启用 → shows "禁用" button; 禁用 → shows "启用" button

### Admin Reset Password Flow

1. Admin clicks "重置密码" in table row
2. `<ResetPasswordDialog>` opens: new password + confirm new password
3. Submit → `POST /api/admin/users/:id/reset-password`
4. **Success:** Toast "已重置 {username} 的密码", dialog closes
