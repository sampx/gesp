    "use client" 组件。共享于学员和管理员设置页。

    表单三个字段（per D-09）：
    - 旧密码：`<Input type="password" placeholder="请输入当前密码" />`
    - 新密码：`<Input type="password" placeholder="请输入新密码（至少6位）" />`
    - 确认新密码：`<Input type="password" placeholder="请再次输入新密码" />`

    验证逻辑：
    - 新密码 ≥6 字符
    - 确认密码与新密码一致
    - 客户端验证失败时 toast.error 提示

    提交逻辑：
    - `fetch POST ${backendUrl}/api/auth/change-password`
    - 携带 cookie: `credentials: "include"` + `headers: { Cookie: "session_id=..." }`
    - 注意：在 client component 中，直接使用 `credentials: "include"` 无法自动发送 httpOnly cookie 到不同端口的后端。需要从 `document.cookie` 提取 session_id。
    - 实际方案：使用 `fetch("/api/auth/change-password", ...)` 通过 NextJS API route 代理，或直接构建完整 URL 并在 headers 中携带 cookie：
      ```typescript
      const sessionId = document.cookie.split(";").find(c => c.trim().startsWith("session_id="))?.split("=")[1];
      const res = await fetch(`${backendUrl}/api/auth/change-password`, {
        method: "POST",
        headers: {
          "Content-Type": "application/json",
          ...(sessionId ? { Cookie: `session_id=${sessionId}` } : {}),
        },
        body: JSON.stringify({ old_password: oldPassword, new_password: newPassword }),
      });
      ```
    - 成功：`toast.success("密码修改成功")`，清空表单（per D-10，不跳转登录页）
    - 失败：`toast.error(data.message || "操作失败")`

    组件 props（可选，用于控制显示样式）：
    ```typescript
    interface PasswordChangeFormProps {
      className?: string;
    }
    ```
