## Critical Issues

### CR-01: FormData Value Null Check 缺失

**File:** `apps/web/src/app/register/page.tsx:42`
**Issue:** `formData.get("username")` 返回值可能为 `null`，直接 `as string` 后调用 `.length` 在极端情况下（如恶意请求绕过浏览器 required 验证）会抛出运行时错误：`TypeError: Cannot read property 'length' of null`

**Fix:**
```typescript
// Line 42 - 修改为安全的 null check
const username = formData.get("username");
if (!username || (username as string).length < 3) {
  toast.error("用户名至少3个字符");
  return;
}
```

**额外建议:** 同文件 Line 29-30 的 `formData.get("password") as string` 和 `formData.get("confirmPassword") as string` 也应添加 null check。

## Warnings

### WR-01: Empty Catch Block 无错误日志

**File:** `apps/web/src/app/register/actions.ts:48-50`
**Issue:** `catch` 块捕获错误后仅返回通用错误消息，未记录实际错误详情，导致生产环境难以排查问题根因。

**Fix:**
```typescript
// Line 48-50 - 添加错误日志（前端可用 console.error）
} catch (err) {
  console.error("Registration failed:", err);
  return { error: "注册失败，请稍后重试" };
}
```

**注意:** 后端 AGENTS.md 禁止 console.error，但这是前端 NextJS Server Action，可使用 console.error。若需更严谨，可引入前端日志库。

### WR-02: FormData 类型断言缺少防御性检查

**File:** `apps/web/src/app/register/actions.ts:8-10`
**Issue:** 直接将 `formData.get()` 结果断言为 `string`，假设字段一定存在。若客户端绕过浏览器验证发送空字段，会导致运行时问题。

**Fix:**
```typescript
// Line 8-10 - 添加 null check
const username = formData.get("username");
const password = formData.get("password");
const confirmPassword = formData.get("confirmPassword");

if (!username || !password || !confirmPassword) {
  return { error: "请填写所有必填字段" };
}

if (password !== confirmPassword) {
  return { error: "两次输入的密码不一致" };
}
```
